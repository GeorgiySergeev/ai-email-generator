### FR-02 · Authentication

**Priority:** P0

**Flows:**
1. **Register** (`/register`): email + password + confirm password → Supabase Auth → create `profiles` record → redirect `/dashboard`
2. **Login** (`/login`): email + password → Supabase Auth → redirect `/dashboard`
3. **Logout** (`/logout`): Server Action → Supabase signOut → redirect `/`

**Validations (Zod):**
```typescript
RegisterSchema = {
  email: z.string().email(),
  password: z.string().min(8).max(100),
  confirmPassword: z.string()
  // .refine(passwords match)
}

LoginSchema = {
  email: z.string().email(),
  password: z.string().min(1)
}
```

**Requirements:**
- Form displays inline errors (per field)
- Loading state on the Submit button
- Invalid login → human-readable message (not raw Supabase error)
- Redirect to `/dashboard` if already authenticated

**Acceptance criteria:**
- [ ] Registration creates a Supabase user and a `profiles` record
- [ ] Login sets a session cookie
- [ ] Logout clears session, redirects to `/`
- [ ] Middleware blocks `/dashboard` for unauthenticated users

### FR-03 · Dashboard — Email Generator

**Priority:** P0

**URL:** `/dashboard`

**UI components:**
1. **Subject field** — `<textarea>` or `<input>`, placeholder: "What is your email about?"
2. **Tone selector** — radio/select with options:
   - Professional
   - Casual
   - Formal
   - Friendly
   - Persuasive
3. **Length selector** — radio/tabs with options:
   - Short (≈ 100 words)
   - Medium (≈ 250 words)
   - Long (≈ 500 words)
4. **Generate button** — disabled while subject is empty
5. **Result area** — displays generated text with a copy button
6. **History section** — last N emails generated by the user

**Validation (Zod):**
```typescript
GenerateEmailSchema = {
  subject: z.string().min(3).max(500),
  tone: z.enum(['professional', 'casual', 'formal', 'friendly', 'persuasive']),
  length: z.enum(['short', 'medium', 'long'])
}
```

**Flow:**
```
User fills form → Submit → Server Action → Zod validate → getAIProvider() →
AI generate → Save to generated_emails → Return result → Display
```

**Acceptance criteria:**
- [ ] Generate button disabled when subject is empty
- [ ] Loading state during generation (spinner, disabled form)
- [ ] Result displayed with a "Copy" button
- [ ] AI error shown clearly (no white screen)
- [ ] Result saved to DB and appears in history
- [ ] History shows last 10 emails

### FR-04 · AI Generation

**Priority:** P0

**Provider:** Anthropic Claude (claude-haiku-4-5) or Mock (configured via `AI_PROVIDER` env)

**AI Provider Interface:**
```typescript
type GenerateEmailParams = {
  subject: string
  tone: 'professional' | 'casual' | 'formal' | 'friendly' | 'persuasive'
  length: 'short' | 'medium' | 'long'
  userLocale?: string
}

type GenerateEmailResult = {
  content: string
  tokensUsed?: number
  modelUsed: string
}
```

**Mock Provider:** returns a deterministic response based on `subject + tone + length`, no API latency.

**Claude Provider:**
- System prompt with tone and length instructions
- Temperature: 0.7
- Max tokens: length-dependent (short: 300, medium: 700, long: 1200)
- Falls back to Mock when `ANTHROPIC_API_KEY` is absent

**Library docs:** always fetch via Context7 MCP → `anthropic` before modifying provider code.

**Acceptance criteria:**
- [ ] Mock provider returns valid email text
- [ ] Claude provider generates with correct tone
- [ ] Switching providers via env requires zero code changes
- [ ] API errors are caught and shown to the user

### FR-07 · Error Handling

**Priority:** P0

**Rules:**
- No white screens (fatal error → `error.tsx` boundary with "Retry" button)
- All API errors logged server-side with context
- User sees a human-readable message (never a stack trace)
- 404 page (`not-found.tsx`)
- Network errors → toast notification

**Error types:**
```typescript
type AppErrorCode =
  | 'AUTH_REQUIRED'     // No session
  | 'AUTH_INVALID'      // Wrong email/password
  | 'AI_UNAVAILABLE'    // AI API unavailable
  | 'RATE_LIMITED'      // Request limit exceeded
  | 'VALIDATION_ERROR'  // Invalid data
  | 'NOT_FOUND'         // Resource not found
  | 'SERVER_ERROR'      // Internal error
```

**Acceptance criteria:**
- [ ] `error.tsx` with Reset button works
- [ ] `not-found.tsx` with a link to home
- [ ] AI errors show a toast, form stays intact
- [ ] Auth errors show an inline message in the form

## 3. Non-Functional Requirements

### NFR-01 · Performance

- First Contentful Paint (FCP) ≤ 1.5s
- Largest Contentful Paint (LCP) ≤ 2.5s

- HTTPS everywhere (Vercel enforced)

- WCAG 2.1 AA minimum

- English (primary)
- Russian (secondary)
- Language switcher in header
- Selection persisted in localStorage/cookie

## 4. User Stories

### Unauthenticated Visitor

```
US-01: As a visitor, I want to see the Landing Page
       so I understand what the service does
       → Open /, see hero + features + demo + faq + pricing

US-02: As a visitor, I want to try generation without registering
       so I can evaluate quality
       → Demo section: enter a subject, see a sample email (mock)

US-03: As a visitor, I want to register
       so I get full access
       → /register → form → redirect /dashboard

US-04: As a visitor, I want to log in
       so I can return to my emails
       → /login → form → redirect /dashboard
```

### Authenticated User

```
US-05: As a user, I want to generate an email from a subject
       so I save time writing
       → Dashboard → enter subject → select tone/length → Generate → see result

US-06: As a user, I want to copy the generated email
       so I can paste it into my email client
       → Copy button next to the result

US-07: As a user, I want to see my email history
       so I can find previously generated emails
       → History list on Dashboard or separate /history page

US-08: As a user, I want to update my name in my profile
       so I can personalise my account
       → /profile → Edit → Save

US-09: As a user, I want to log out
       so I can protect my data
       → Logout button → redirect /

US-10: As a user, I want to see subscription plans
       so I understand how to get more features
       → /pricing → see Free / Pro / Enterprise
```

- No avatar upload (first letter of name only)
- History: read-only, no search/filter
