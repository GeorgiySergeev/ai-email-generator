## What Is This Project

**MVP AI Email Generator** — a SaaS application built on Next.js 15.  
The user enters an email subject, selects a tone and length, and receives a ready-to-use email body generated by an LLM.

## 30-Second Architecture

```
Next.js 15 (App Router) + TypeScript + Tailwind 4 + shadcn/ui
        │
        ├── (marketing)/ ── Landing Page, Pricing ── Server Components
        ├── (auth)/      ── Login, Register ── Server Actions + Supabase Auth
        └── (dashboard)/ ── Dashboard, Profile, History ── Protected + TanStack Query
                │
                ├── Server Actions → Zod validation → AI Provider → Supabase
                └── Zustand (UI state) + Server Actions (server mutations)

AI Provider Pattern:
  getAIProvider() → env AI_PROVIDER=mock  → MockProvider
                 → env AI_PROVIDER=claude → ClaudeProvider
```

## Top 10 Rules (full rules in RULES.md)

1. **Functional only** — no classes (except ErrorBoundary)
2. **TypeScript strict** — no `any`, no `@ts-ignore` without reason
3. **Tailwind CSS** — no inline styles, no hardcoded colors/spacing (tokens only)
4. **Naming:** `camelCase` variables, `PascalCase` components, `SCREAMING_SNAKE_CASE` constants
5. **Server Components by default** — `'use client'` only when necessary
6. **Zod everywhere** — all input data validated at system boundaries
7. **`cn()` for conditional classes** — not template literals or string concatenation
8. **Mobile-first** — start with base styles, add `sm:`, `md:`, `lg:` breakpoints
9. **Named exports** for all components except page/layout
10. **`ActionResult` types** for all Server Actions: `{ success: true; data } | { success: false; error }`
11. **Context7** — fetch library docs before using any API or installing packages

## Common Mistakes — Avoid These

1. Do not add `'use client'` to Server Components without necessity
2. Do not hardcode colors in Tailwind: `text-[#3B82F6]` → use `text-primary`
3. Do not use `useEffect` for data fetching — use Server Components or Server Actions
4. Do not mutate state directly in Zustand — use `set()`
5. Do not forget `await` before `createClient()` from `@supabase/ssr` (it's async in App Router)
6. Do not use `router.push()` after Server Action — use `redirect()` from `next/navigation` on the server
7. Do not create Supabase client in every component — use shared clients from `src/lib/supabase/`
8. Do not write library code from memory — always use Context7 first
